# Document Workflow

## Overview
This workflow creates comprehensive documentation for Python projects including README, API docs, and guides.

## Steps

### 1. Understand the Audience
- Identify target users (developers, end users)
- Determine appropriate technical level
- Choose documentation style

### 2. Gather Information
- Read the code
- Understand functionality
- Identify key features

### 3. Structure Documentation
- Create outline
- Organize sections logically
- Add examples

### 4. Write and Refine
- Write clear, concise content
- Add code examples
- Review and improve

## Usage
```
/document API reference
/document user guide
/document architecture
```

## Output
- README file
- API documentation
- User guides
- Developer guides
- Architecture documentation
